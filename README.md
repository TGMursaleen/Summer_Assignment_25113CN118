## 💡 Self-Reflection: Weekly Challenges & My Solutions

### 🔴 Week 1 Challenges (C Logic & Recursion)
*   **The GCD Block (Formula & Logic Challenge):** 
    *   **The Problem:** While writing code for Greatest Common Divisor (GCD) on Day 3, I hit a wall because I didn't know the mathematical formula or Euclidean algorithm required to break down the problem into an efficient loop.
    *   **How I Handled It:** I searched on Google, understood the Euclidean relation between division remainders and common factors, and implemented the iterative formula to solve the question.
*   **The Day 7 Transition (Recursion):** 
    *   **The Problem:** Shifting from iterative loops to recursive call stacks exactly on the day I switched my entire codebase to Java syntax was an overwhelming jump. 
    *   **How I Handled It:** I started sketching "Dry Run Trees" on paper for recursive functions (like Fibonacci). Breaking down the execution into a clear base case and recursive case helped me visualize how Java resolves functions from memory.

### 🔵 Week 2 Challenges (Pattern Layouts & Array Buffer Grinds)
*   **The Problem (Nested Loop Spaces):** In Day-10 pyramids, aligning spaces alongside incrementing numbers/characters required deep calculation of row-column dependencies[cite: 2].
*   **The Toughest Part (Array Crash Control):** During Day-14 array exercises, programs crashed instantly with `InputMismatchException` when passing alphabets to break the loops, or arrays sorted extra initialized blocks (`0`s) using generic `Arrays.sort()`[cite: 2].
*   **How I Handled It:** 
    1. For patterns, I formulated linear equations relating rows to spaces ($spaces = total\_rows - current\_row$)[cite: 2].
    2. For array bugs, I integrated advanced java strategies like clearing the scanner engine buffer via `sc.next()` and scoping my array sorts dynamically using `Arrays.sort(arr, 0, count)` to process only live data[cite: 2].
